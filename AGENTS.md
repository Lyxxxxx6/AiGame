# Repository Guidelines

- 必须使用中文与我对话。

# 角色设定 / 项目要求
你是一位非常资深的游戏制作人 + HTML5 Canvas / 原生 JavaScript 高级工程师，
精通高速反馈、爽感节奏、程序化生成、美术冲击设计，
擅长在不依赖任何外部库的情况下，手写物理、赛道、特效与音效系统。
你对爆燃型游戏美术风格有极高造诣，能用极简技术实现强烈视觉冲击。
--------------------------------------------------------------------------------
任务
编写一个单文件 HTML5 手机网页游戏，名为：
《失控赛车：不刹车》
游戏在pc上进行游玩，通过 和控制赛车方向。
--------------------------------------------------------------------------------
核心玩法概述
类型
极速竞速 / 极限反应 / 爽感失控 / 短局爆燃游戏
--------------------------------------------------------------------------------
核心规则（不可变）
- 赛车永远无法减速
- 速度只会不断增加
- 玩家只能左右移动 / 漂移
- 结局必然是失控爆炸
--------------------------------------------------------------------------------
完整单局流程设计（必须完整实现）
 开局：3 秒上头（强启动）
- 倒计时：3  2  1  GO  
- 赛车瞬间冲出起点
- 速度数值立刻飙升
- 镜头前倾 + 强烈速度线
- 引擎音高频拉升
👉 玩家第一秒就明确感知：我已经停不下来了
--------------------------------------------------------------------------------
 中段：持续加压（失控递增）
- 速度随时间持续指数增长  
- 障碍密度、车流密度同步上升  
- 玩家只能靠 极限左右滑动 生存
系统持续制造差点撞上的爽点：
- 贴边擦过障碍 / 车辆
- 触发 Near Miss（险过）
- 获得分数与能量奖励
--------------------------------------------------------------------------------
 超车击杀系统（核心爽点之一）
超车
- 高速接近前车进入超车判定区
- 成功贴脸超越：
- 触发闪光
- 音效爆点
- 速度短暂提升
击杀
- 在极高速下从侧后方撞击敌车
- 敌车发生：
- 夸张爆炸
- 车体碎片飞散
- 玩家获得：
- 分数
- Fever 能量
- 瞬间加速
👉 玩法从「躲避」升级为「主动猎杀」
--------------------------------------------------------------------------------
 爽点循环（每 10 秒一个高潮）
赛道中周期性触发事件：
- 高速加速带（音爆 + 镜头震动）
- 密集车流穿梭区
- 高速漂移急弯（短暂子弹时间）
--------------------------------------------------------------------------------
 Fever 模式：彻底失控
触发条件：
连续险过 / 超车击杀累计
进入 Fever 后：
- 速度进入超载状态
- 画面扭曲、色彩爆闪
- 赛车带残影
- 普通障碍可被撞飞
- 音效进入失真轰鸣
👉 从被压迫  碾压世界
--------------------------------------------------------------------------------
 终局：爆炸式死亡（强记忆点）
结束条件：
- 撞击次数耗尽或  
- 速度超过极限阈值强制失控
死亡表现必须包含：
- 画面冻结 0.1 秒  
- 镜头极速拉近  
- 巨型爆炸  
- 全屏粒子喷射  
- 屏幕震动 + 低频音爆
结算界面：
- 最终速度  
- 生存距离  
- 险过次数  
- 超车击杀数  
- Fever 次数  
- 巨大的「再来一局」按钮
--------------------------------------------------------------------------------
赛道程序化生成系统（必须实现）
- 无尽赛道，实时生成，多车道，车道中间会有障碍物
- 参数驱动：
- 弯道曲率
- 障碍密度
- 车流频率
- 难度随时间与速度递增
- 支持随机事件：
- 车阵
- 路面变化
- 环境切换（城市 / 荒漠 / 夜景）
- 障碍物密度、车辆密度、车速控制 等等这些核心参数 暴露出来，供玩家可调试
--------------------------------------------------------------------------------
美术风格设计（爆燃级）
整体风格定位
赛博朋克  末日竞速  高饱和爆燃风
关键词：
- 高对比
- 强光效
- 快速形变
- 失控感
--------------------------------------------------------------------------------
赛车视觉
- 未来机械感造型
- 高亮描边
- 加速时拉伸（Squash & Stretch）
- 撞击时局部变形
--------------------------------------------------------------------------------
赛道与环境
- 伪 3D 透视赛道
- 障碍物由远及近快速放大
- 霓虹广告牌、废墟、灯带
- Fever 时背景整体扭曲
相机视角
玩家视角为车后方的第三视角，接近第一视角的角度，更有爽感。
--------------------------------------------------------------------------------
特效（Juice）
- 火花
- 爆炸碎片
- 氮气喷射
- 速度残影
- 屏幕震动
--------------------------------------------------------------------------------
音效要求（必须）
- 禁止使用音频文件
- 使用 Web Audio API
- 程序化合成：
- 引擎持续音
- 加速音爆
- 碰撞低频轰鸣
- 音高与速度强绑定
--------------------------------------------------------------------------------
代码要求
- 单文件（HTML + CSS + JS）
- Canvas 渲染
- 原生 JS，不依赖库
- 全局错误捕捉：
- try-catch
- window.onerror
- 红色屏幕调试台
--------------------------------------------------------------------------------
UI 要求
顶部实时显示：
- 当前速度
- 险过连击
- Fever 能量条
- 剩余生命
UI 风格：
- 半透明
- 霓虹描边
- 数值跳动有动画
--------------------------------------------------------------------------------
核心设计哲学（不可违背）
速度永远在增加
玩家永远在边缘
死亡必须震撼
再来一局必须无法抗拒
